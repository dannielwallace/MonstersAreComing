# PLAN.md - 行城守望 开发计划

> 目标游戏：[Monsters Are Coming! Rock & Road](https://store.steampowered.com/app/2934220/Monsters_are_Coming/) (Steam App ID: 2934220)

---

## 目标游戏核心玩法解析

《Monsters Are Coming! Rock & Road》是一款 **Tower-Survivor Roguelite** 游戏，核心特色：

| 特性 | 说明 |
|------|------|
| **移动行城** | 城市沿道路自动前进，玩家必须持续防御追逐的怪物群 |
| **双重角色** | 英雄可跳下行城砍怪、伐木、采集资源；同时管理行城防御塔 |
| **roguelite 升级** | 击杀敌人获得宝石/经验，选择升级英雄或行城 |
| **波次+Boss** | 怪物波次进攻，Boss 战具有召唤机制 |
| **资源建设** | 采集资源→建造/升级防御→推进到下一区域 |
| **风格** | 中世纪卡通风格，2D 俯视角 |

**来源：** [CGMagazine Review](https://www.cgmagonline.com/review/game/monsters-are-coming-rock-road/) · [Inverse](https://www.inverse.com/gaming/monsters-are-coming-tower-defense-survival) · [Rock Paper Shotgun](https://www.rockpapershotgun.com/monsters-are-coming-turns-boring-skill-upgrades-into-urban-planning-with-deadly-stakes) · [GameSpew](https://www.gamespew.com/2025/11/monsters-are-coming-rock-road-review/)

## MVP 阶段划分

### MVP 0 ✅ 已完成：灰盒原型（行城 + 防御塔 + 波次敌人）
- 核心数据结构 + 渲染
- 战斗系统 + 波次导演

### MVP 1 🔄 当前阶段：可玩核心循环（让游戏"好玩"）

让游戏从"能跑"变成"好玩"——加入移动地图、英雄离城、视觉反馈。

| # | 任务 | 预估 | 说明 |
|---|------|------|------|
| 1.1 | 行城自动前进 + 地图滚动 | 中 | 行城沿 x 轴自动前进，相机跟随，地图元素（树、石）从右侧不断生成 |
| 1.2 | 英雄离城采集机制 | 中 | 英雄可以在行城周围自由移动采集，靠近资源自动采集 |
| 1.3 | 资源节点可视化 + 碰撞采集 | 小 | 树木（棕色圆+绿色树冠）、石矿（灰色方块），靠近时自动采集 |
| 1.4 | 敌人血条 + 行城血条 | 小 | 敌人头顶小血条，行城 HUD 血条可视化 |
| 1.5 | 建筑可视化升级 | 中 | 箭塔=三角形图标，投石车=圆形图标，城墙=矩形+砖纹 |
| 1.6 | 投射物 + 命中特效 | 中 | 箭=细长直线飞行，投石=抛物线，命中=闪烁/粒子 |

| 1.8 | 升级弹出 UI 美化 | 小 | 三选一卡片式弹出，带图标和描述 |

**MVP 1 成功标准：** 一个陌生人能在不看说明的情况下玩 5 分钟并理解"采集→建造→防守→升级"的循环。

## 本次迭代：MVP 1 详细计划

### Task 1.1: 行城自动前进 + 地图滚动

**文件：** `GameScene.ts`

**改动：**
- 移除 WASD 移动行城的逻辑（行城改为自动前进）
- 行城以恒定速度（~40px/s）沿 x 轴前进
- 相机跟随行城（`camera.startFollow(caravan)`）
- 资源节点（树、石）在行城前方动态生成
- 超过相机视野的资源自动回收

### Task 1.2: 英雄离城采集

**文件：** `GameScene.ts`, 新增 `src/game/hero.ts`

**改动：**
- WASD 控制英雄（不再是行城）
- 英雄可脱离行城自由移动
- 靠近资源节点（~30px）时自动开始采集
- 采集有动画延迟（每棵树木 2s 砍完）

### Task 1.3: 资源节点可视化

**文件：** `GameScene.ts`

**改动：**
- 树木：棕色树干 + 绿色圆形树冠
- 石矿：灰色方块
- 采集进度条（悬浮在资源上方）

### Task 1.4: 血条系统

**文件：** `GameScene.ts`, `src/game/combat.ts`

**改动：**
- 每个敌人头顶添加小型血条（绿→红渐变）
- 行城 HUD 血条改为可视化条形进度条
- 城墙血条（悬浮在城墙上方，仅受损时显示）

**文件：** `GameScene.ts`

## 开发流程（PDCA 闭环）

每个任务遵循：
1. **Plan** — 读取现有代码，确定改动点
2. **Do** — 编写代码，同时写/更新测试
3. **Check** — `npm test` + `npm run build` → Playwright 截图验证
4. **Act** — 修复 bug，重新测试
5. **Commit** — Conventional Commit 格式提交
