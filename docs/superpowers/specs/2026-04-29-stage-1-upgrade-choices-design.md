# Stage 1 Upgrade Choices Design

## 目标

为当前 Phaser 灰盒原型加入第一版 Roguelite 成长循环：敌人死亡后给予经验，经验满时暂停战斗并展示 3 个中文升级选项，玩家选择后立即强化当前局内能力并恢复战斗。

本阶段的目标不是做完整养成系统，而是验证“战斗压力 -> 获得经验 -> 三选一强化 -> 继续防守行城”的核心节奏是否清晰、有反馈、可扩展。

## 范围

包含：

- 敌人死亡时给予经验值。
- HUD 显示当前等级和经验进度。
- 经验达到阈值时提升 1 级，并进入升级选择状态。
- 升级选择状态暂停敌人移动、生成、攻击、采集和箭塔开火。
- 屏幕中央展示 3 个升级选项，全部玩家可见文案使用简体中文。
- 玩家可以按 `1`、`2`、`3` 选择升级。
- 支持鼠标点击升级选项，作为键盘选择的补充。
- 选择升级后应用属性效果，关闭面板并恢复战斗。
- 新增纯 TypeScript 模块承载经验、升级池、升级抽取和属性应用规则。
- 新增单元测试覆盖经验升级、三选一抽取和升级属性效果。
- 保持 `npm test` 和 `npm run build` 通过。

不包含：

- 稀有度、升级颜色品质、复杂权重、升级重随、弃选、暂停菜单、永久养成、存档、成就、音效、动画特效、新怪物、新建筑、Boss、完整美术资源。

## 玩家体验

玩家继续围绕行城采集木材、建造箭塔并抵御怪物。敌人被箭塔击杀后，HUD 的经验进度上涨。经验满时，场上行动冻结，中央出现 3 张升级选项。

玩家选择其中一个后，效果立刻生效，例如箭塔打得更远、更疼、射得更快，或采集木材更快、行城生命上限更高。随后战斗恢复，玩家能在接下来的压力中感受到成长差异。

第一版升级应偏“数值直接可感知”，避免复杂条件触发。玩家不需要理解构筑体系，只需要知道“杀怪会升级，选项会让当前局更强”。

## 经验规则

- 初始等级为 1。
- 初始经验为 0。
- 每个普通敌人死亡给予固定经验 `5`。
- 升级所需经验使用简单递增阈值：`20 + (level - 1) * 12`。
- 如果一次获得经验超过当前等级阈值，保留溢出经验。
- 第一版每次只弹出一个升级选择面板；如果理论上连续升多级，下一次升级面板在当前选择完成后再弹出。
- 游戏结束后按 `R` 重开时，等级、经验、升级效果和升级选择状态全部重置。

## 升级池

第一版升级池使用轻量属性升级，全部为局内效果。

首批升级：

- `箭塔校准`：箭塔射程提升 20。
- `重弩箭头`：箭塔伤害提升 5。
- `快速装填`：箭塔攻击间隔降低 12%，但最低不低于 0.25 秒。
- `伐木熟手`：采集速度提升 25%。
- `坚固车体`：行城最大生命提升 20，并立即回复 20 点生命，但当前生命不能超过新的最大生命。
- `前线修补`：立即回复行城 25 点生命，但当前生命不能超过最大生命。

抽取规则：

- 每次升级从可用升级池中抽取 3 个不同选项。
- 如果升级池少于 3 个可用选项，则展示全部可用选项；当前首批升级池不少于 3 个。
- 第一版允许同一升级在不同等级重复出现和重复选择。
- 同一次三选一内不能出现重复选项。
- 抽取可以使用 Phaser/浏览器随机数，但纯逻辑模块应支持注入随机函数，便于测试。

## 局内属性

为支持升级，当前固定常量需要被运行时属性承载。

新增运行属性：

- `towerRange`
- `towerDamage`
- `towerFireInterval`
- `gatherRate`
- `caravanMaxHealth`

保留既有 Stage 0 默认值作为初始属性：

- 箭塔射程：190
- 箭塔伤害：10
- 箭塔攻击间隔：0.55 秒
- 采集速度：8 木材/秒
- 行城最大生命：100

这些属性只影响新一轮运行中的即时表现，不写入任何持久化数据。

## UI 与交互

HUD 新增：

- `等级：1`
- `经验：0/20`

升级弹窗：

- 标题：`选择升级`
- 副标题：`按 1 / 2 / 3 选择`
- 每个选项显示中文名称和中文效果说明。
- 面板应居中显示，覆盖在游戏场景上方。
- 背景可使用半透明深色遮罩，让玩家明确当前处于暂停选择状态。
- 选项可以用简单矩形卡片实现，保持灰盒风格。

输入规则：

- 升级面板打开时，`1`、`2`、`3` 只用于选择升级。
- 升级面板打开时，`Space` 不建造箭塔，`WASD` 不移动玩家。
- 鼠标点击选项时，与对应数字键效果一致。
- 游戏结束状态优先于升级选择；正常情况下升级选择不会在游戏结束后弹出。

## 暂停规则

升级选择状态应冻结会改变战局的系统：

- 玩家移动暂停。
- 行城移动暂停。
- 敌人移动暂停。
- 敌人生成暂停。
- 敌人攻击暂停。
- 箭塔攻击暂停。
- 采集暂停。
- 运行计时暂停，避免升级选择时间影响生存时长。

渲染可以继续显示当前场景，不需要隐藏敌人、塔或资源。

## 架构

新增模块建议：

- `src/game/experience.ts`
  - 负责经验阈值、经验增加、等级提升和溢出经验计算。
  - 不依赖 Phaser。
- `src/game/upgrades.ts`
  - 定义升级 ID、中文名称、中文描述、效果类型和数值。
  - 负责抽取 3 个不重复升级。
  - 负责把升级效果应用到运行属性。
  - 不依赖 Phaser。

`GameScene.ts` 负责：

- 在敌人死亡时调用经验规则。
- 根据经验结果打开升级面板。
- 管理升级选择状态。
- 渲染 HUD 和升级面板。
- 将数字键和点击事件映射到升级选择。
- 在选择后应用升级并恢复更新循环。

避免把升级规则直接写死在 Phaser 场景里。场景可以组织 UI 和生命周期，但升级数值和抽取规则应保持可测试。

## 错误处理与边界

- 如果没有待选升级，面板不应打开，游戏继续运行。
- 如果玩家按下无效数字键，忽略输入，不关闭面板。
- 如果升级面板已打开，新的经验变化不应覆盖当前 3 个选项。
- 如果敌人死亡导致升级，同时行城生命已经归零，游戏结束优先，不打开升级面板。
- 快速连续按键或点击只应应用一次升级。
- 重开游戏必须销毁旧升级面板和旧点击交互对象。

## 测试

自动化测试：

- `requiredExperienceForLevel(1)` 返回 20。
- 更高等级所需经验递增。
- 获得不足阈值经验时不升级。
- 获得达到阈值经验时等级提升并保留溢出经验。
- 升级抽取返回 3 个不重复选项。
- 注入固定随机函数时，升级抽取结果可预测。
- 应用 `箭塔校准` 后箭塔射程增加。
- 应用 `重弩箭头` 后箭塔伤害增加。
- 应用 `快速装填` 后攻击间隔降低且不低于下限。
- 应用 `伐木熟手` 后采集速度提升。
- 应用 `坚固车体` 后最大生命和当前生命按规则增加。
- 应用 `前线修补` 后当前生命回复但不超过最大生命。

手动验证：

- 击杀敌人后 HUD 经验上涨。
- 经验满后游戏冻结并显示中文升级面板。
- 面板展示 3 个不同升级选项。
- 按 `1`、`2`、`3` 能选择对应升级。
- 点击卡片能选择对应升级。
- 选择后面板消失，战斗恢复。
- 所选升级在后续战斗中有可见效果。
- 升级面板打开时不能移动、采集或建造。
- 按 `R` 重开后等级、经验和升级效果恢复初始值。
- `npm test` 通过。
- `npm run build` 通过。

## 验收标准

本阶段完成时：

- 游戏内存在“击杀敌人 -> 获得经验 -> 升级三选一 -> 应用强化”的完整闭环。
- 所有新增玩家可见升级 UI 文案均为简体中文。
- 升级选择状态能清楚暂停当前战局，选择后能恢复。
- 首批至少 5 个升级可用，且每次展示 3 个不重复选项。
- 经验和升级规则位于可单测的纯 TypeScript 模块中。
- 不引入新美术、新音频或引用原作受版权保护素材。
- `npm test` 和 `npm run build` 通过。
